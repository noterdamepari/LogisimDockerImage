# LogisimDockerImage
#### 🛠 Требования
1. Docker (инструкция по установке ниже)
2. make
3. git
#### 🐳 Установка Docker
Наиболее простым и быстрым вариантом является установка Docker Desktop по данной ссылке:
https://www.docker.com/get-started/

Если же у вас Linux, то рекомендуется устанавливать cli версию через ваш пакетный менеджер.
##### Подробная установка на MacOS:
Рекомендуется ставить docker через пакетный менеджер brew. Чтобы поставить brew введите в терминал команду:<br>
`/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"`<br>
Для проверки установки можно использовать `brew -v`. Вывод должен быть примерно такой **Homebrew 5.0.14**

Далее для установки Docker используем команду:<br>`brew install —cask docker`<br> После установки надо зайти в приложение Docker.app и выбрать пункт с рекомендованной настройкой, чтобы docker установил нужные ему зависимости. 

Для проверки установки Docker можно использовать команду: `docker -v`.  Вывод должен быть примерно такой **Docker version 29.2.0, build 0b9d198**

#### 🏗️ Cборка
Создайте папку и склонируйте туда репозиторий, далее командой: `make build-container` соберите образ.️

‼️️Tак как образ сам скачивает и устанавливает JVM, ему может потребоваться больше ресурсов чем обычно, если вы используете Docker Desktop, рекомендуется поставить более высокие лимиты ресурсов.
#### ⚙️ Подготовка к запуску
Откройте файл `config.mk`<br>
1. В переменной `LOGISIM_PORT` укажите удобный для вас порт (по стандарту 5800)<br>
2. Создайте на вашем пк папку, посредством которой будет осуществляться обмен файлами с образом<br>
В переменной `LOGISIM_VOLUME` укажите путь до созданной вами папки. (по стандарту создается папка workspace в директории куда вы склонировали репозиторий)
3. Для лучшего отображения в переменных `DISPLAY_WIDTH` и `DISPLAY_HEIGHT` укажите разрешение вашего экрана (по стандарту FullHD 1920x1080) 
#### 🚀 Запуск
Для запуска контейнера используйте команду: `make up-logisim`
После запуска в командной строке отобразится адрес, по которому нужно перейти в браузере для получения доступа к самому приложению
#### 🖥 Взаимодействие с образом
Для получения/отправки схем в образ используйте папку, которую указали в конфиге. Схемы будут автоматически синхронизироваться.<br>
В образе файлы будут находиться в директории `/workspace` <br>

Для выключения образа используйте команду: `make down-logisim` <br>
Для проверки текущего состояния: `make status-logisim`
#### 💡 Рекомендации по использованию
Весь проект разрабатывался и тестировался на браузере Firefox, в других браузерах может работать нестабильно, поэтому к использованию рекомендуется именно он.
#### ❗ Известные проблемы
1. Комбинации клавиш работают только на английской раскладке<br>
2. На MacOS перепутаны cmd и ctrl
